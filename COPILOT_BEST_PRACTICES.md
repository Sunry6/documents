# 🚀 VS Code + GitHub Copilot 编程最佳实践指南

AI 辅助编程正在改变我们构建软件的方式。Copilot 已经不仅仅是一个“高级代码补全工具”，而是一个可以结对编程的 Agent。为了充分发挥 VS Code + Copilot 的潜力，我们总结了以下最佳实践。

## 1. 🧠 谋定而后动：先计划，后编码

**痛点**：直接让 AI 写复杂功能，它往往会“瞎猜”上下文，写出一堆需要反复修改的废代码。
**最佳实践**：在让 Copilot 动手写代码前，先让它输出**实施计划**。

- **操作方法**：在 Copilot Chat 中输入类似以下的 Prompt：
  > _"我要实现 [某功能]，请先分析代码并给我一个详细的实施计划（包含需要修改的文件路径和核心逻辑步骤）。在我回复‘同意’之前，请不要直接生成代码。"_
- **优势**：这迫使 AI 先理清思路，也让你有机会在它写代码前纠正错误的方向。如果计划偏了，**不要试图通过对话去修补代码，而是直接修改计划**，重新生成。

## 2. 🎯 精准投喂：掌控上下文 (Context)

AI 的输出质量 100% 取决于你提供的上下文。不要指望 AI 能自动猜透整个庞大的代码库。

- **善用 `@` 和 `#` 变量**：
  - `@workspace`：当你不确定相关代码在哪时，使用 `@workspace 这里的鉴权流程是怎么样的？`，让 Copilot 全局检索。
  - `#file` 和 `#selection`：如果你明确知道要改哪几个文件，直接在聊天框输入 `#` 引入特定文件。**精准引入比全局搜索效果更好，能有效减少“上下文噪音”。**
  - `@terminal`：终端报错时，直接问 `@terminal 为什么报错，如何修复？`，它会自动读取终端的错误日志。
- **果断开启新对话 (New Chat)**：
  - **何时新建**：开始新任务、发现 Copilot 开始“胡言乱语”或反复犯错时。过长的对话会积累噪音，导致 AI 失去焦点。
  - **何时继续**：迭代同一个功能、需要刚才讨论的上下文时。

## 3. 📜 沉淀团队规范：配置专属 Rules

不要每次都在对话里重复“请使用 TypeScript”、“请用 Tailwind CSS”。你可以为项目配置全局指令，让 Copilot 永远遵守团队规范。

- **操作方法**：在项目根目录创建 `.github/copilot-instructions.md` 文件（或在 VS Code 设置中搜索 `Copilot Instructions`）。
- **内容示例**：

  ```markdown
  # 代码风格

  - 优先使用 React 函数组件和 Hooks。
  - 样式使用 Tailwind CSS，不要写内联样式。
  - 所有的组件必须有完整的 TypeScript 类型定义。

  # 工作流

  - 优先使用 pnpm 运行命令。
  - 编写测试时，请使用 Jest 框架。
  ```

- **优势**：Copilot 在生成代码时会自动读取这个文件作为系统提示词（System Prompt），确保输出风格统一。

## 4. 🔄 常用高效工作流

- **测试驱动开发 (TDD)**：
  - 步骤 1：_"我要实现一个工具函数，请先帮我写 Jest 测试用例，涵盖边界情况。"_
  - 步骤 2：运行测试，确认失败。
  - 步骤 3：_"测试已失败，请帮我编写实现代码，使其通过测试。"_
  - **优势**：有了明确的测试目标，AI 生成的代码准确率会大幅提升。
- **快速理解陌生代码**：
  - 面对祖传代码，直接选中代码并提问：_"这段逻辑处理了哪些边缘情况？"_ 或 _"为什么这里要调用 `setUser` 而不是 `createUser`？"_
- **Git 自动化**：
  - 写完代码后，在 Source Control 面板点击 Copilot 图标（✨），它会自动阅读 `git diff` 并生成规范的 Commit Message。

## 5. 🐛 科学排障：复刻 Debug 模式

遇到诡异 Bug（如竞态条件、性能问题）时，**不要直接问“怎么修”**，AI 往往会给出错误的猜测。

- **最佳实践**：让 AI 帮你“打日志”和“提假设”。
  > _"我遇到了一个偶发的白屏 Bug，请提出 3 个可能导致该问题的假设，并帮我在关键位置加上 `console.log`。我会运行代码并把日志贴给你分析。"_
- **视觉排障**：VS Code 的 Copilot Chat 支持**拖拽图片**。你可以直接把 UI 设计图扔给它让它写组件，或者把浏览器的报错截图发给它排查。

## 6. 💡 驾驭 AI 的核心心法

能从 Copilot 中获益最多的开发者，通常具备以下特质：

1. **提示词极其具体**：对比一下 _"加个测试"_ 和 _"为 auth.ts 编写一个覆盖注销边缘情况的测试用例，避免使用 mock"_，后者的成功率是前者的十倍。
2. **极其仔细地审查代码**：AI 生成的代码可能看起来很完美，但暗藏细微的逻辑漏洞。**AI 写得越快，你的 Code Review 就越重要。**
3. **提供可验证的目标**：AI 无法修复它不知道的问题。配置好 Linter、TypeScript 和自动化测试，给 AI 提供明确的“对错信号”。
4. **把 AI 当作平等的协作者**：向它索要计划，要求它解释思路，并果断拒绝你认为糟糕的架构方案。
