# VS Code + GitHub Copilot 公司化落地信息模板

> 目的：把公司已有 instruction/规范沉淀为“可执行的 Copilot 最佳实践”。
> 使用方式：回公司电脑后，按下面清单填写（可脱敏），发给我即可生成最终版文档。

## 1) Instruction 体系（最重要）

### 1.1 文件位置与优先级

- Instruction 文件都在哪些目录（例如 `.github/`、`.cursor/`、`docs/standards/`）
- 是否有优先级规则（例如：项目规则 > 团队规则 > 个人规则）
- 是否区分前端/后端/数据/测试子项目规则

### 1.2 关键规则摘要（可复制原文片段）

- 代码风格（命名、目录结构、注释、模块边界）
- 工程规范（TypeScript 严格模式、lint、format、测试覆盖率）
- 安全与合规（密钥、PII、日志、审计、依赖安全）
- 提交与评审（commit/PR 模板、必须检查项）

---

## 2) 技术栈与项目现实约束

- 主要技术栈（React/Vue、Node、Java、Go 等）
- 包管理与构建（pnpm/npm/yarn、CI 平台）
- 必跑命令（例如 `lint`、`typecheck`、`test`、`build`）
- 常见限制（例如禁止引入新依赖、禁止跨目录改动）

---

## 3) 你们现在真实的 Copilot 使用方式

- 日常场景 Top 5（写功能、改 Bug、写测试、读代码、写文档）
- 当前痛点 Top 5（写偏、幻觉、改动过大、风格不统一、上下文污染）
- 你希望文档优先解决的问题（按优先级排序）

---

## 4) 团队流程约束（把 AI 纳入 SDLC）

- 需求到上线流程（需求 -> 开发 -> 自测 -> PR -> 发布）
- AI 允许做什么 / 禁止做什么
- 高风险改动的人工 Gate（安全、鉴权、数据迁移）
- 回滚与应急策略

---

## 5) 提示词与产出模板偏好

- 你希望统一的 Prompt 模板字段（目标、约束、上下文、验收、输出格式）
- 你希望统一的“变更报告”格式（变更点、风险、验证结果、回滚方案）
- 是否需要中英文双语模板

---

## 6) 脱敏与安全边界（必填）

- 不可外传信息清单（内部域名、客户名、业务规则等）
- 可分享级别（完全公开 / 仅公司内部 / 仅团队内部）
- 需要我在文档里加入的“信息脱敏守则”

---

## 7) 样例材料（至少提供 3 项）

- 1~2 个“好例子”：Copilot 输出很成功的对话/PR
- 1~2 个“坏例子”：Copilot 走偏的典型案例
- 1 个高价值模块的真实任务描述（可脱敏）

---

## 8) 交付偏好

- 文档受众：新同事 / 资深工程师 / Tech Lead / 全员
- 期望长度：短版（2-3 页）/ 标准版（5-8 页）/ 手册版（10+ 页）
- 是否要附录：常用 Prompt 库、Review 清单、FAQ

---

## 推荐你回公司后最小收集包（MVP）

1. instruction 文件列表 + 每个文件一句话作用
2. 3~5 个关键规范文件（可脱敏）
3. 你们 CI 必跑命令
4. 2 个成功案例 + 1 个失败案例

> 有了上面四项，我就可以生成第一版“公司定制版 Copilot 最佳实践”。
